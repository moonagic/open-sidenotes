# agent.md

This file is a working memory for Codex in this repository.
It captures current product direction and collaboration rules.

## Product Direction

- Keep Open Sidenotes as a notes-first app for fast memo capture.
- Continue improving AI chat as a core capability next to notes.
- Remove/de-emphasize task-module style UX from the main product flow.
- Prioritize a minimal, elegant, low-friction UI over feature-heavy layouts.

## UX Priorities

- Settings must stay easy to reach from both Notes and AI Chat contexts.
- Avoid deep navigation chains for common actions.
- Keep mode switching obvious and one-step when possible.
- Reduce visual noise; prefer clean spacing, clear hierarchy, and simple controls.

## Engineering Preferences

- Refactors are allowed when they improve structure, reliability, and UX.
- Prefer iterative but substantial improvements over tiny cosmetic-only changes.
- Keep the app buildable after each meaningful batch of edits.

## Collaboration Rules

- Codex may commit to git proactively without asking for approval each time.
- Commit when a logical, testable milestone is complete.
- Use clear commit messages describing user-visible outcomes.
